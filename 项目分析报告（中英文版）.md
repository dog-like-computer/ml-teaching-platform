# Project Analysis Report

## 1. Project Overview

This project is a machine learning model training and deployment platform designed to provide users with a complete workflow from data upload, preprocessing, model training to model deployment and SDK generation. The platform supports various machine learning algorithms, including classification, regression, and clustering task types.

### Core Features
- User authentication and management (registration, login, password reset)
- Data upload, management, and preprocessing
- Model training and evaluation
- Model prediction API
- SDK generation and download

## 4. Core Module Analysis

### 4.1 Main Application Class (App)

The main application class `App` is the core of the entire system, responsible for handling HTTP requests, route management, and server startup.

**Main Functions:**
- Route registration and management
- HTTP request processing
- Response construction
- Database connection initialization
- Training worker management

**Key Methods:**
- `route()`: Route decorator for registering API routes
- `handle_request()`: Handles HTTP requests
- `match()`: Matches routes and calls corresponding processing functions
- `start()`: Starts the server and training workers

### 4.2 Training Worker (TrainWorker)

The `TrainWorker` class is responsible for processing background model training tasks.

**Main Functions:**
- Fetching training tasks from the queue
- Creating model instances and training
- Evaluating model performance
- Generating SDKs
- Updating database status

**Key Methods:**
- `process_task()`: Processes a single training task
- `_create_model()`: Creates model instances
- `_generate_sdk()`: Generates SDK packages
- `run()`: Runs the worker to process tasks in the queue

### 4.3 Database Operations (Mysql)

The `Mysql` class provides database operation encapsulation, supporting asynchronous database operations.

**Main Functions:**
- Database query
- Data insertion
- Data update
- Data deletion
- Direct SQL statement execution

**Key Methods:**
- `find()`: Queries data
- `insert()`: Inserts data
- `update()`: Updates data
- `delete()`: Deletes data
- `execute()`: Executes SQL statements

### 4.4 Data Processing (Panda)

The `Panda` class encapsulates data processing-related functions based on the pandas library.

**Main Functions:**
- File reading (CSV, Excel)
- Data preprocessing (deduplication, filling missing values, handling outliers)
- Text vectorization
- Data normalization
- Data format conversion

### 4.5 Encryption Tools (Other)

The `Other` class provides encryption and random ID generation functions.

**Main Functions:**
- Password encryption
- Random ID generation
- Training method retrieval

## 8. Key Technical Points

The system uses `asyncio` and `aiomysql` to implement asynchronous processing, improving concurrent performance. Especially when handling database operations and network requests, asynchronous processing can significantly improve system response speed.

### 8.2 Model Training Queue

The system uses Redis as a task queue to implement asynchronous processing of model training. When users submit training tasks, the tasks are added to the queue, and background workers fetch tasks from the queue and execute them.

### 8.3 SDK Generation

The system can generate deployable SDKs for trained models, including model files, configuration files, and usage examples. The SDK supports local prediction and batch prediction functions, and provides a PyQt5 interface.

### 8.4 Data Preprocessing
